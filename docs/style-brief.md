# Style Brief

## Design intent
Build a site that feels premium, calm, and intentional for service-business buyers, but still looks like it was made by a strong frontend developer with taste. The goal is not "creative for its own sake." The goal is immediate trust plus obvious craft.

## Core principles
- High-trust first: clean hierarchy, fast comprehension, polished spacing, clear calls to action.
- Bold in controlled ways: typography, layout rhythm, color blocking, and motion should carry the distinction.
- Personal, not agency-generic: the site should feel like a sharp solo operator with a point of view.
- Editorial over template: sections should feel composed, not assembled from a startup landing page kit.
- Mobile quality is non-negotiable: the mobile layout must feel intentionally designed, not compressed from desktop.

## Hard bans
- No generic SaaS purple-to-blue gradients on white.
- No cookie-cutter hero with centered headline, pills, and fake dashboard cards unless the concept truly demands it.
- No default font stacks or common "AI-made" typography pairings.
- No random glassmorphism, glow spam, floating orbs, or meaningless abstract decoration.
- No dense grids of identical cards without a clear reason.
- No trying to impress through sheer animation volume.

## Typography direction
- Prefer one expressive display voice and one restrained working voice.
- Lean toward a refined serif-plus-sans or high-character grotesk-plus-serif pairing.
- Favor contrast, rhythm, and readable line length over trendy type choices.
- Candidate directions:
  - elegant serif display + quiet neo-grotesk sans
  - compact editorial serif + sturdy workhorse sans
  - sharp grotesk headline + warm text serif for contrast
- Any chosen type system should feel deliberate on desktop and stay legible on mobile.

## Color logic
- Base palette should stay light-first.
- Use warm neutral or soft mineral backgrounds instead of pure white.
- Main text should sit in an ink, deep navy, charcoal, or similarly grounded dark tone.
- Use one assertive accent color. It should feel chosen, not default.
- Accent should direct attention to CTA moments, separators, and key emphasis, not cover the whole page.

## Layout rules
- Favor asymmetry, strong vertical pacing, and clear section-to-section mood shifts.
- Use generous whitespace where it improves clarity, not because "luxury" is the default answer.
- Let one or two moments on the page break the grid on purpose.
- Keep line lengths comfortable and content blocks scannable.
- Every section needs a job: explain, prove, reassure, or convert.

## Motion rules
- Use restrained but meaningful motion.
- Prefer entrance choreography, hover confidence, and subtle parallax or transforms over constant movement.
- Animation should support hierarchy and polish, not compete with content.
- Honor reduced-motion preferences.

## Component rules
- Buttons should feel precise and intentional, not default Tailwind buttons.
- Cards, borders, dividers, and badges need a reason to exist.
- Proof sections should use hierarchy and framing, not generic testimonial or metric tiles.
- Forms should feel calm and easy to complete.

## Content and visual balance
- Copy should remain the primary signal. Visual design supports it.
- Use mockups, screenshots, diagrams, and process visuals sparingly and with strong framing.
- When in doubt, remove decoration and improve composition.

## Final quality bar
If a section looks like it could have come from any AI landing-page generator, it fails. The finished page should feel specific to this business, this buyer, and this offer set.
