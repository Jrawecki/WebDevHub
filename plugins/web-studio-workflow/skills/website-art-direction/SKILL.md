# Website Art Direction

This skill is the project authority for design taste on the Web Dev Biz Site. When it is relevant, use it before coding or revising any major section.

## Read these first
- `../../../../docs/style-brief.md`

Project-local guidance overrides generic external design skills when they conflict.

## What this skill must enforce
- The site must feel high-trust, premium, and intentional.
- The design must not resemble a stock SaaS template or AI-generated landing page.
- Every page needs a visible concept, not just tasteful defaults.
- Mobile layouts must feel composed, not squeezed down from desktop.

## Hard bans
- Generic SaaS gradients, especially purple-on-white defaults
- Template-looking hero sections
- Default font stacks or overused AI-era type pairings
- Random glassmorphism, glow effects, or ornamental shapes without purpose
- Repetitive card grids that flatten hierarchy

## Required moves
- Commit to a clear visual concept before implementation.
- Use typography as a primary design tool.
- Create hierarchy that is obvious within five seconds.
- Use motion sparingly, but make it feel deliberate and polished.
- Give at least one or two moments per page a memorable composition choice.

## Working method
- Start by naming the visual concept in a short phrase.
- Design around the buyer and offer, not around trends.
- Prefer removing weak decorative ideas over keeping them.
- If a section looks familiar in a bad way, redesign it until it feels specific to this business.

## Review gate
Reject or revise any design that:
- could plausibly come from a generic AI website generator
- looks more like a startup app than a service-business studio
- has weak CTA emphasis or muddy hierarchy
- relies on decoration instead of composition and copy
